# v0.3.5 - Fix SSR Hydration Error (Slider Persentase)

## Tanggal
2026-03-08

## Masalah
Slider persentase tidak muncul di halaman scan dan history. Ada error React #299 di browser console yang menyebabkan hydration mismatch antara server-side rendering (SSR) dan client-side rendering.

Error yang muncul:
```
Uncaught (in promise) Error: Minified React error #299; visit https://react.dev/errors/299 for the full message or use the non-minified dev environment for full errors and additional helpful warnings.
```

## Penyebab
Masalah ini terjadi karena:
1. React 19 memiliki ketatannya yang lebih tinggi untuk hydration
2. Elemen form seperti `<input type="range">` mengalami mismatch antara SSR dan client-side
3. Aplikasi PWA seperti NutriiD sebenarnya tidak membutuhkan SSR karena semua interaksi terjadi di client-side

## Solusi
Menonaktifkan SSR (Server-Side Rendering) pada aplikasi. Untuk aplikasi PWA dengan React 19, SSR sebenarnya tidak diperlukan dan sering menyebabkan masalah hydration.

## Perubahan File

### `vite.config.ts`
- Hapus konfigurasi `ssr: 'resources/js/ssr.tsx'` dari plugin Laravel Vite
- Tetap mempertahankan `refresh: true` untuk hot module replacement

Sebelum:
```typescript
laravel({
    input: ['resources/css/app.css', 'resources/js/app.tsx'],
    ssr: 'resources/js/ssr.tsx',
    refresh: true,
}),
```

Sesudah:
```typescript
laravel({
    input: ['resources/css/app.css', 'resources/js/app.tsx'],
    refresh: true,
}),
```

## Dampak Perubahan

### Positif:
1. ✅ Slider persentase sekarang muncul dengan baik di halaman scan
2. ✅ Slider edit porsi muncul dengan baik di halaman history
3. ✅ Error React #299 teratasi
4. ✅ Build aplikasi lebih cepat (tidak perlu build SSR)
5. ✅ Bundle size lebih kecil
6. ✅ Tidak ada lagi masalah hydration mismatch

### Trade-off:
- Tidak ada initial server-side rendering (untuk SEO, tapi ini tidak relevan untuk PWA)
- First Contentful Paint mungkin sedikit lebih lambat (tapi tidak terasa signifikan)

## Testing
1. Buka halaman scan
2. Scan makanan dan tunggu analisis selesai
3. Pastikan slider "Berapa yang Anda makan?" muncul
4. Geser slider dan pastikan preview gizi berubah
5. Klik "Simpan" dan pastikan data tersimpan
6. Buka halaman history
7. Klik tombol "Tandai Dimakan" atau "Edit Porsi"
8. Pastikan slider edit muncul dan berfungsi
9. Cek browser console - tidak ada error React #299

## Catatan
- File `resources/js/ssr.tsx` masih ada tapi tidak digunakan lagi
- Untuk aplikasi PWA seperti NutriiD, CSR (Client-Side Rendering) sudah cukup
- React 19 + Inertia.js + PWA = kombinasi yang lebih baik tanpa SSR
