# v0.3.4 - Add Daily Summary Refresh on History Page Actions

## Tanggal
2026-03-08

## Masalah
Di halaman riwayat (history.tsx), ketika user mengedit porsi makanan atau menghapus scan, komponen DailySummary tidak ter-refresh secara otomatis. Hal ini menyebabkan data ringkasan hari ini tidak sinkron dengan data yang sebenarnya.

## Solusi
Menambahkan mekanisme refresh menggunakan `refreshKey` untuk memastikan DailySummary selalu menampilkan data terbaru setelah aksi edit atau delete.

## Perubahan File

### `resources/js/components/daily-summary.tsx`
- Tambah properti `refreshKey?: number` pada interface `DailySummaryProps`
- Tambah `refreshKey` sebagai parameter fungsi component
- Tambah `refreshKey` ke dependency array `useEffect` sehingga komponen akan memanggil `fetchDailySummary()` saat `refreshKey` berubah
- Ubah format persentase dari desimal ke integer menggunakan `Math.floor(progress)` untuk tampilan yang lebih bersih (misal: 100% bukan 100,00%)

### `resources/js/pages/history.tsx`
- Tambah state baru `summaryRefreshKey` dengan nilai awal 0
- Di fungsi `deleteScan`: Tambah `setSummaryRefreshKey((prev) => prev + 1)` setelah berhasil menghapus scan
- Di fungsi `saveEdit`: Tambah `setSummaryRefreshKey((prev) => prev + 1)` setelah berhasil menyimpan edit porsi
- Update komponen `<DailySummary />` menjadi `<DailySummary refreshKey={summaryRefreshKey} />`

## Cara Kerja

### Flow Edit Porsi:
1. User klik "Tandai Dimakan" atau "Edit Porsi" pada scan
2. User atur persentase konsumsi
3. User klik "Simpan"
4. `saveEdit()` dipanggil â†’ PUT ke `/api/food-scan/{id}` â†’ Update consumed_percentage di database
5. State `summaryRefreshKey` di-increment (misal dari 0 ke 1)
6. Komponen `DailySummary` menerima prop `refreshKey` baru
7. `useEffect` ter-trigger karena dependency `refreshKey` berubah
8. `fetchDailySummary()` dipanggil ulang untuk mendapatkan data terbaru
9. Ringkasan hari ini ter-update

### Flow Delete Scan:
1. User klik ikon sampah pada scan
2. Konfirmasi delete
3. `deleteScan()` dipanggil â†’ DELETE ke `/api/food-scan/{id}` â†’ Hapus record dari database
4. State `summaryRefreshKey` di-increment (misal dari 1 ke 2)
5. Komponen `DailySummary` menerima prop `refreshKey` baru
6. `useEffect` ter-trigger karena dependency `refreshKey` berubah
7. `fetchDailySummary()` dipanggil ulang untuk mendapatkan data terbaru
8. Ringkasan hari ini ter-update

## Catatan
- Penggunaan `refreshKey` adalah pattern umum di React untuk memaksa re-render atau refetch data
- Setiap kali `summaryRefreshKey` di-increment, React menganggapnya sebagai perubahan prop yang memicu useEffect
- Pendekatan ini lebih efisien daripada me-reload seluruh halaman
