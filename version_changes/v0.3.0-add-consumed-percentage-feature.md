## Perubahan Alur Pengguna

### Sebelumnya
1. User scan makanan
2. AI menganalisis
3. Hasil langsung disimpan ke riwayat dan dihitung ke daily summary

### Sekarang
1. User scan makanan
2. AI menganalisis
3. Hasil ditampilkan dengan slider persentase (default 0%)
4. User dapat mengatur persentase yang dimakan (0-100%)
5. User klik "Simpan" untuk menyimpan ke riwayat
6. Jika persentase > 0, gizi dihitung ke daily summary berdasarkan persentase
7. Di halaman riwayat, user dapat mengedit persentase kapan saja

## Fitur Baru

1. **Slider Persentase Makanan**
   - Input slider 0-100% untuk menentukan porsi yang dimakan
   - Preview gizi yang akan dihitung ke daily summary
   - Label "Hanya cek gizi" (0%) dan "Dimakan sepenuhnya" (100%)

2. **Edit Porsi di Riwayat**
   - Setiap scan di riwayat memiliki tombol "Tandai Dimakan" atau "Edit Porsi"
   - Slider untuk mengubah persentase
   - Preview gizi yang akan dihitung ke daily summary
   - Badge status "Dimakan X%" untuk visualisasi

3. **Perhitungan Daily Summary yang Akurat**
   - Daily summary hanya menghitung makanan dengan `consumed_percentage > 0`
   - Perhitungan gizi: `gizi_asli * (consumed_percentage / 100)`

## Catatan

- Semua scan tetap disimpan ke riwayat, tidak ada yang hilang
- Scan dengan `consumed_percentage = 0` hanya untuk cek gizi (tidak dihitung ke daily summary)
- Scan dengan `consumed_percentage > 0` dihitung ke daily summary berdasarkan persentase
- User dapat mengedit consumed_percentage kapan saja melalui halaman riwayat
- Daily summary menggunakan backend calculation yang sudah terupdate, frontend tidak perlu perubahan
