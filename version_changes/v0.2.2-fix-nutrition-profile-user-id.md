# v0.2.2 - Fix Nutrition Profile User ID Error

## Date
2026-03-08

## Summary
Fixed database error when creating new nutrition profile records. The `user_id` field was not included in the model's `$fillable` array, preventing mass assignment during record creation.

## Problem Solved

**Error**: `SQLSTATE[HY000]: General error: 1364 Field 'user_id' doesn't have a default value`

**Root Cause**: When users tried to save their nutrition profile for the first time, Laravel's mass assignment protection blocked the `user_id` field because it wasn't listed in the `$fillable` array of the `NutritionProfile` model.

**Impact**: Users could not create new nutrition profile records. The system would throw a database error when attempting to insert a record without a `user_id` value.

## Changes Made

### Models
- **Updated**: `NutritionProfile.php`
  - Added `'user_id'` to the `$fillable` array
  - This allows the `user_id` field to be mass-assigned during model creation

### Technical Details

The `updateOrCreateForUser()` method in `NutritionProfile` uses:
```php
return self::create(array_merge(['user_id' => $userId], $data));
```

Without `'user_id'` in the `$fillable` array, Laravel's mass assignment protection would strip this field from the data array, causing the database to reject the insert due to a missing required field.
