# Version 0.1.2 - Fix Camera Display Issue

## Date: 2026-03-04

## Issue
Camera not displaying on `/scan` page - only showing loading spinner.

## Root Cause
Circular dependency in the video rendering logic:
- Video element was only rendered when `isStreaming` was true
- `isStreaming` was only set to true after video metadata loaded and play() succeeded
- But video metadata couldn't load because video element wasn't in the DOM (since `isStreaming` was false)

## Fix
**File**: `resources/js/pages/scan.tsx`

Changed the conditional rendering logic:
- Video element now always renders when there's no captured image
- Added `opacity` style to hide video until streaming starts
- Loading spinner shows as overlay while camera initializes
- Camera switch button only shows when streaming is active

### Before:
```tsx
{isStreaming && !capturedImage ? (
    <video ref={videoRef} ... />
) : capturedImage ? (
    <img ... />
) : (
    <Loader2 />
)}
```

### After:
```tsx
{!capturedImage ? (
    <>
        <video ref={videoRef} style={{ opacity: isStreaming ? 1 : 0 }} ... />
        {!isStreaming && <Loader2 overlay />}
        {isStreaming && <SwitchCamera button />}
    </>
) : (
    <img ... />
)}
```

## Technical Notes
- Video element must be in DOM for `videoRef.current` to exist
- Stream can only be attached to video element that exists in DOM
- Using opacity instead of conditional render ensures video element is always available for stream attachment
