# v0.5.7 - Fix PHP Maximum Execution Time Error

## Problem
The nutrition analysis feature was returning a 500 Internal Server Error. The error logs showed:
```
Maximum execution time of 30 seconds exceeded
```

## Root Cause
- PHP's default `max_execution_time` is set to 30 seconds
- The HTTP request to OpenRouter API has a 120-second timeout
- PHP was killing the script before the API call could complete
- This happened in `NutritionAnalysisService.php` when making the AI API call

## Solution
Added `set_time_limit(180)` at the beginning of the `analyzeDailyNutrition()` method in `app/Services/NutritionAnalysisService.php` to extend the execution time limit to 3 minutes, which is sufficient for the AI API call to complete.

## Changes Made
- Modified `app/Services/NutritionAnalysisService.php`
  - Added `set_time_limit(180)` to increase execution time to 3 minutes

## Impact
- Nutrition analysis feature now works correctly
- API calls to OpenRouter have enough time to complete
- Users can successfully analyze their nutrition data with AI
