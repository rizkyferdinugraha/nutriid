# Version 0.6.7 - Fix Desktop Dark Mode Default

## Changes
- Added `--color-scheme: light` to Tailwind CSS `@theme` configuration
- Forces light mode to be the default across all pages, including landing page
- Prevents Tailwind CSS v4 from automatically applying dark mode based on system preferences

## Problem
Even though the landing page had all dark mode classes removed, Tailwind CSS v4 was still applying dark mode on desktop systems that had dark mode enabled in system preferences. This caused the landing page to appear in dark mode despite the light mode design.

## Solution
By setting `--color-scheme: light` in the `@theme` block, we explicitly tell Tailwind CSS to use light mode as the default across the entire application, overriding any system preference detection.
