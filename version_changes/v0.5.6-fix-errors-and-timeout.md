# v0.5.6 - Fix Error Handling and Timeout Issues

## Date
2026-03-09

## Changes

### 1. Fix API Timeout Issue (NutritionAnalysisService.php)
**Problem:** Nutrition analysis API calls were failing with "Maximum execution time of 30 seconds exceeded" error when calling OpenRouter API.

**Solution:** Increased the timeout from 60 seconds to 120 seconds in `NutritionAnalysisService.php`:
- Changed `->timeout(60)` to `->timeout(120)` in the HTTP request

**File Modified:** `app/Services/NutritionAnalysisService.php`

### 2. Fix Root Element Not Found Error (app.tsx)
**Problem:** React/Inertia was showing "Root element not found" error in console during page initialization.

**Solution:** Improved error handling in `app.tsx`:
- Added better logging to help diagnose hydration issues
- Added check for DOM ready state
- Added informative console messages when root element is missing

**File Modified:** `resources/js/app.tsx`

### 3. Improve Error Handling in Controller (NutritionAnalysisController.php)
**Problem:** Generic error messages didn't help users understand what went wrong with AI analysis requests.

**Solution:** Added specific exception handling in `NutritionAnalysisController.php`:
- Added `ConnectionException` handler for network issues
- Added `RequestException` handler for API request issues
- Improved error messages in Indonesian

**File Modified:** `app/Http/Controllers/NutritionAnalysisController.php`

## Impact
- **Timeout Issue Resolved:** Users can now successfully complete nutrition analysis without timeout errors
- **Better Error Messages:** Users receive clear, actionable error messages when something goes wrong
- **Improved Debugging:** Enhanced console logging helps diagnose initialization issues
