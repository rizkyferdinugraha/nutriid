# v0.3.6 - Perbaiki Slider Live Update di Halaman Riwayat

## Tanggal
2026-03-08

## Perubahan
Memperbaiki bug di halaman riwayat (history.tsx) di mana nilai nutrisi pada bagian "Akan dihitung ke daily summary" tidak berupdate secara langsung ketika slider digeser.

## Detail Teknis

### Masalah
- Ketika pengguna mengedit porsi makanan di halaman riwayat, slider berfungsi tetapi nilai nutrisi yang ditampilkan (kalori, protein, karbohidrat, lemak) tidak berupdate secara real-time
- Nilai nutrisi tetap menampilkan berdasarkan `scan.consumed_percentage` yang lama, bukan `editingPercentage` yang sedang diubah

### Solusi
- Memodifikasi perhitungan `actualNutrition` di dalam loop rendering
- Menambahkan conditional logic: jika dalam mode edit (`isEditing`), hitung nutrisi berdasarkan `editingPercentage`, jika tidak gunakan `scan.consumed_percentage`
- Menggunakan spread operator untuk membuat temporary object dengan percentage yang benar sebelum dikirim ke fungsi `getActualNutrition`

### File yang Diubah
- `resources/js/pages/history.tsx`

## Impact
- Pengguna sekarang dapat melihat perubahan nilai nutrisi secara langsung (real-time) ketika menggeser slider untuk mengubah porsi makanan
- Meningkatkan UX karena pengguna dapat melihat preview sebelum menyimpan perubahan
- Format persentase sekarang ditampilkan tanpa desimal (misalnya "50%" bukan "50,00%") pada:
  - Nilai persentase di slider saat edit
  - Badge "Dimakan X%" pada card makanan
  - Pesan notifikasi toast setelah menyimpan
