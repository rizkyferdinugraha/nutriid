# v0.3.8 - Add Complete Description & Failed Scan Detection

## Release Date
2026-03-08

## Changes

### Database
- **New Migration** (`database/migrations/2026_03_08_103841_add_scan_status_to_food_scans_table.php`)
  - Added `scan_status` field to `food_scans` table
  - Enum values: `success` or `failed`
  - Default value: `success`

### Models
- **FoodScan Model** (`app/Models/FoodScan.php`)
  - Added `scan_status` to fillable fields

### AI Service
- **Updated AI Prompt** (`app/Services/AIService.php`)
  - Changed from brief description (1-2 sentences) to complete description (3-5 sentences)
  - Added specific instructions for description content: food type, main ingredients, cooking method, appearance, and notable features
  - Added example of comprehensive description in Indonesian
  - Improved description quality and detail level
  - Added automatic scan status detection based on confidence score
  - Scans with confidence score < 30% are marked as `failed`

### Backend Controller
- **FoodScanController** (`app/Http/Controllers/FoodScanController.php`)
  - Updated to save `scan_status` from AI analysis results

### Frontend Pages
- **Scan Page** (`resources/js/pages/scan.tsx`)
  - Added `scan_status` to TypeScript interface
  - Shows warning toast when scan status is `failed`
  - Warning message: "⚠️ Identifikasi Rendah - Makanan kurang jelas. Data gizi mungkin tidak akurat"

### Frontend Components
- **Nutrition Card** (`resources/js/components/nutrition-card.tsx`)
  - Added `scan_status` to TypeScript interface
  - Added `AlertTriangle` icon import
  - Added warning banner for failed scans

- **Scan Detail Modal** (`resources/js/components/scan-detail-modal.tsx`)
  - Added `scan_status` to TypeScript interface
  - Added `AlertTriangle` icon import
  - Added warning banner for failed scans

## Impact
- Users will now see more detailed and informative descriptions of scanned food
- Descriptions provide better context about the food including ingredients, preparation, and characteristics
- Failed scan detection helps users identify when nutrition data may be inaccurate
- Visual warnings alert users to rescan or verify data manually
- Improved user experience with better readability and visual presentation
- No breaking changes - backward compatible with existing scans

## Technical Details
- AI model response time: No significant change
- Description length: Increased from ~10-20 words to ~50-80 words (3-5 sentences)
- Scan status threshold: Confidence score < 30% = failed
- UI components: Enhanced styling for better readability and failed scan indicators
- AI Service timeout: Increased from 30 to 60 seconds to handle API delays
- Error handling: Added user-friendly timeout messages in Indonesian
- Logging: Enhanced error logging for debugging scan failures
