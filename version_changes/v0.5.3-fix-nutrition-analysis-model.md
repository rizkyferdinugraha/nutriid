# Version v0.5.3 - Fix Nutrition Analysis Model

## Summary
Fixed the AI model used in nutrition analysis to use text-only models instead of vision models, which were causing 500 errors when analyzing daily nutrition without images.

## Changes Made

### File: `app/Services/NutritionAnalysisService.php`

**Issue:** The `NutritionAnalysisService` was using vision models (designed for image analysis) for text-only nutrition analysis, causing API errors when the endpoint expected text input but received vision model requirements.

**Fix:** Updated the `availableModels` array to use text-only models:

```php
// OLD (vision models - caused errors):
protected array $availableModels = [
    'qwen/qwen3-vl-235b-a22b-thinking',
    'mistralai/mistral-small-3.1-24b-instruct:free',
    'qwen/qwen3-vl-30b-a3b-thinking',
    'nvidia/nemotron-nano-12b-v2-vl:free',
];

// NEW (text-only models):
protected array $availableModels = [
    'qwen/qwen-2.5-72b-instruct',
    'mistralai/mistral-small-3.1-24b-instruct:free',
    'qwen/qwen-2.5-32b-instruct',
    'google/gemma-2-27b-it:free',
    'meta-llama/llama-3.2-3b-instruct:free',
];
```

## Rationale

The nutrition analysis feature analyzes text data (daily nutrition summary, user profile, food items) and doesn't require vision capabilities. Using text-only models:

1. **Prevents API errors** - Text-only models are optimized for text input and won't fail expecting image data
2. **Better performance** - Text-only models are faster and more efficient for this use case
3. **Cost efficiency** - Many text-only models have free tiers available
4. **Matches use case** - The analysis is purely based on structured nutrition data, not images

## Models Selected

- **Primary:** `qwen/qwen-2.5-72b-instruct` - High-quality instruction-following model
- **Free options:** 
  - `mistralai/mistral-small-3.1-24b-instruct:free`
  - `google/gemma-2-27b-it:free`
  - `meta-llama/llama-3.2-3b-instruct:free`

## Testing

The nutrition analysis endpoint should now work correctly:
- POST `/api/nutrition-analysis/analyze` will use text-only models
- No more 500 Internal Server Error
- Response will include: general advice, important info, health recommendations, and nutrition status
