# v0.7.0 - Fix Desktop Dark Mode Nutrition Cards

## Date
2026-03-10

## Changes
- Removed all `dark:` classes from nutrition card components to prevent dark backgrounds on desktop
- Fixed nutrition cards, shapes, and badges that were displaying with dark backgrounds on desktop mode
- Despite dark mode being removed in v0.6.6, some `dark:` classes remained causing background to appear dark on systems with dark mode enabled

## Files Modified
1. `resources/js/components/nutrition-card.tsx` - Removed dark: classes from nutrition card macros
2. `resources/js/components/daily-summary.tsx` - Removed dark: classes from daily summary stats and progress bar
3. `resources/js/pages/analysis.tsx` - Removed dark: classes from macro nutrients and advice sections
4. `resources/js/components/scan-detail-modal.tsx` - Removed dark: classes from nutrition cards and warning boxes
5. `resources/js/pages/history.tsx` - Removed dark: classes from nutrition badges and consumed status

## Implementation Details
- Changed background classes from e.g., `bg-orange-50 dark:bg-orange-950/30` to just `bg-orange-50`
- Changed gradient backgrounds from e.g., `bg-gradient-to-br from-orange-50 to-orange-100 dark:from-orange-950/30 dark:to-orange-900/20` to just `bg-gradient-to-br from-orange-50 to-orange-100`
- Changed text classes from e.g., `text-amber-900 dark:text-amber-100` to just `text-amber-900`
- Changed border classes from e.g., `border-amber-200 dark:border-amber-900` to just `border-amber-200`

## Impact
- Nutrition cards, shapes, and badges now display with light backgrounds consistently on all devices
- Desktop users with system dark mode enabled will no longer see dark backgrounds on nutrition elements
- Maintains clean, light-mode visual experience across all screen sizes

## Technical Notes
- The issue was caused by Tailwind CSS `dark:` classes that were not removed when dark mode feature was removed in v0.6.6
- On systems with dark mode enabled, these classes would still apply, causing backgrounds to appear dark
- All affected components now use light-mode classes only, matching the light-only design system
