# v0.6.2 - Fix White Screen Issue (SSR & Appearance Variables)

## Date
2026-03-10

## Description
Fixed white screen issue that prevented application from loading any content.

## Issue Details

**Symptoms:**
- Application displayed only a white screen with no error messages
- No content visible on any page
- Browser console showed no obvious errors

**Root Causes:**
1. **SSR Issue:** Inertia SSR (Server Side Rendering) was enabled in `config/inertia.php` and configured to connect to `http://127.0.0.1:13714`, but the SSR server was not running. This caused all page requests to fail silently.
2. **Undefined Variable:** Blade template referenced undefined `$appearance` variable which could cause rendering errors
3. **Debug Mode:** `APP_DEBUG=false` suppressed error messages, making it difficult to diagnose

## Changes Made

### config/inertia.php
- Changed `'enabled' => true` to `'enabled' => false` in SSR configuration
- Disabled SSR to prevent white screen when SSR server is not running
- Added inline comment explaining the change

### .env
- Changed `APP_ENV=production` to `APP_ENV=local`
- Changed `APP_DEBUG=false` to `APP_DEBUG=true`
- Enabled debug mode to show actual errors for easier troubleshooting

### resources/views/app.blade.php
- Removed `$appearance` variable reference from HTML tag `@class` directive
- Fixed inline JavaScript to use simplified appearance logic
- Changed from dynamic dark mode detection to fixed light mode
- Updated script to set `colorScheme: 'light'` directly

### Assets Rebuilt
- Executed `npm run build` to regenerate all asset files
- New asset hashes: `app-BX1sw5Vb.css`, `app-DBq1co4r.js`, etc.

### Cache Cleared
- Executed `php artisan config:clear` to clear configuration cache
- Executed `php artisan cache:clear` to clear application cache

## Technical Details

**Why This Fixes the Issue:**
1. Inertia was attempting to render pages server-side via SSR, but the SSR server was unavailable
2. When the SSR server at `http://127.0.0.1:13714` was not running, requests failed
3. In production mode with `APP_DEBUG=false`, errors were suppressed
4. This resulted in a completely blank page with no error messages
5. Additionally, the undefined `$appearance` variable in the blade template could cause rendering issues

**Client-Side Rendering (CSR):**
- With SSR disabled, Inertia falls back to client-side rendering
- This is the default and most common mode for Inertia.js applications
- The application now loads normally without requiring a separate SSR server

**Appearance Simplification:**
- Removed dynamic dark mode detection since the app is light-mode only
- Simplified the inline script to set fixed light mode
- Eliminated potential errors from undefined variables

**Future SSR Implementation:**
If SSR is desired in the future for SEO or performance:
1. Start the SSR server: `npm run build:ssr` then `node bootstrap/ssr/ssr.mjs`
2. Update `config/inertia.php` to set `'enabled' => true`
3. Ensure the SSR server URL matches the running server
4. Or use Laravel's built-in SSR bundle configuration instead of URL
