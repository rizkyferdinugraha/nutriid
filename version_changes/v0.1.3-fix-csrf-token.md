# Version 0.1.3 - Fix CSRF Token for Food Scan API

## Date
2026-03-04

## Issue
- Food scan API returning 419 (CSRF token mismatch) error when analyzing images
- The frontend code was trying to get CSRF token from meta tag that didn't exist

## Changes
- Added `<meta name="csrf-token" content="{{ csrf_token() }}">` to `resources/views/app.blade.php`

## Files Modified
- `resources/views/app.blade.php` - Added CSRF token meta tag in head section

## Technical Details
Laravel requires CSRF token verification for all POST requests. The frontend JavaScript in `scan.tsx` was correctly trying to include the token:
```javascript
'X-CSRF-TOKEN': document.querySelector('meta[name="csrf-token"]')?.getAttribute('content') || ''
```

But the meta tag was missing from the HTML template, causing the token to be empty string, which resulted in 419 error.
