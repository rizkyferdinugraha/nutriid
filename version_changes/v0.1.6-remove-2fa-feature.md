# v0.1.6 - Hapus Fitur Autentikasi 2 Faktor

## Tanggal: 8 Maret 2026

## Deskripsi
Menghapus seluruh fitur Autentikasi 2 Faktor (Two-Factor Authentication) dari aplikasi NutriID.

## File yang Dihapus

### Komponen Frontend
- `resources/js/components/two-factor-recovery-codes.tsx` - Komponen kode pemulihan 2FA
- `resources/js/components/two-factor-setup-modal.tsx` - Modal pengaturan 2FA

### Hooks
- `resources/js/hooks/use-two-factor-auth.ts` - Hook untuk manajemen 2FA

### Pages
- `resources/js/pages/settings/two-factor.tsx` - Halaman pengaturan 2FA
- `resources/js/pages/auth/two-factor-challenge.tsx` - Halaman challenge 2FA saat login

### Routes
- `resources/js/routes/two-factor/index.ts` - Route utama 2FA
- `resources/js/routes/two-factor/login/index.ts` - Route login 2FA

### Actions (Generated)
- `resources/js/actions/Laravel/Fortify/Http/Controllers/TwoFactorAuthenticatedSessionController.ts`
- `resources/js/actions/Laravel/Fortify/Http/Controllers/TwoFactorAuthenticationController.ts`
- `resources/js/actions/Laravel/Fortify/Http/Controllers/TwoFactorQrCodeController.ts`
- `resources/js/actions/Laravel/Fortify/Http/Controllers/TwoFactorSecretKeyController.ts`
- `resources/js/actions/Laravel/Fortify/Http/Controllers/ConfirmedTwoFactorAuthenticationController.ts`
- `resources/js/actions/Laravel/Fortify/Http/Controllers/RecoveryCodeController.ts`

### Backend Controller
- `app/Http/Controllers/Settings/TwoFactorAuthenticationController.php` - Controller pengaturan 2FA

### Tests
- `tests/Feature/Auth/TwoFactorChallengeTest.php` - Test challenge 2FA
- `tests/Feature/Settings/TwoFactorAuthenticationTest.php` - Test pengaturan 2FA

## File yang Diubah

### Configuration
- `config/fortify.php` - Menghapus `Features::twoFactorAuthentication()` dari konfigurasi features

### Layouts
- `resources/js/layouts/settings/layout.tsx` - Menghapus menu "Autentikasi 2 Faktor" dari sidebar navigasi

### Actions Index
- `resources/js/actions/Laravel/Fortify/Http/Controllers/index.ts` - Menghapus import dan export controller 2FA

## Database

### Migration Baru
- `database/migrations/2026_03_08_004950_remove_two_factor_columns_from_users_table.php` - Menghapus kolom 2FA dari tabel users:
  - `two_factor_secret`
  - `two_factor_recovery_codes`
  - `two_factor_confirmed_at`

### Migration Lama (Tetap Ada)
- `database/migrations/2025_08_14_170933_add_two_factor_columns_to_users_table.php` - File migration asli (tidak dihapus untuk menjaga riwayat)

## Fitur yang Dihapus
- Autentikasi dua faktor saat login
- Pengaturan 2FA di menu settings
- QR code untuk setup 2FA
- Kode pemulihan darurat (recovery codes)
- Kunci manual setup 2FA
- Challenge halaman 2FA
- Rate limiter khusus 2FA

## Fitur yang Tetap Ada
- Login dengan email dan password
- Reset password
- Verifikasi email
- Registrasi user
- Pengaturan profil
- Pengaturan password
- Pengaturan tampilan (dark/light mode)

## Impact pada User
- User yang sebelumnya mengaktifkan 2FA tidak akan lagi diminta memasukkan kode 2FA saat login
- Kolom 2FA di database akan dihapus (data kode 2FA akan hilang)
- Menu "Autentikasi 2 Faktor" di settings tidak lagi muncul
- Proses login menjadi lebih sederhana tanpa step tambahan 2FA

## Alasan Penghapusan
- Simplifikasi proses autentikasi
- Mengurangi kompleksitas aplikasi
- Mengurangi dependensi eksternal (aplikasi autentikator)
- Mempercepat proses login user
- Sesuai dengan kebutuhan aplikasi yang lebih sederhana
