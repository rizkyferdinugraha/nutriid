# v0.4.2 - Fix Vite React Refresh in Production

## Date
2026-03-09

## Description
Fixed an issue where `@viteReactRefresh` was being loaded in production environment, causing the browser to try connecting to the Vite dev server (port 5173) which is not running after build, resulting in `ERR_CONNECTION_REFUSED` errors.

## Changes Made

### resources/views/app.blade.php
- Added environment check to only load `@viteReactRefresh` in `local` and `testing` environments
- Changed from unconditional `@viteReactRefresh` to:
  ```blade
  @if (app()->environment(['local', 'testing']))
      @viteReactRefresh
  @endif
  ```

### public/hot
- Removed `public/hot` file which contained `https://nutriid.test:5173`
- This file was forcing Laravel's Vite plugin to connect to the Vite dev server instead of using built assets
- The file is automatically created when running `npm run dev` and should not exist in production

## Issue Details
**Symptoms:**
- After running `npm run build`, browser console showed errors:
  - `GET https://nutriid.test:5173/@vite/client net::ERR_CONNECTION_REFUSED`
  - `GET https://nutriid.test:5173/resources/js/app.tsx net::ERR_CONNECTION_REFUSED`
  - `GET https://nutriid.test:5173/@react-refresh net::ERR_CONNECTION_REFUSED`

**Root Cause:**
The `@viteReactRefresh` directive was unconditionally loading in all environments, including production. After building, the application still attempted to load resources from the Vite dev server which is no longer running.

**Resolution:**
Conditionally load `@viteReactRefresh` only in development environments (`local`, `testing`). In production, only the built assets in `public/build` are referenced via `@vite()`.

## Testing
- Development mode: Hot module replacement should work normally
- Production build: No more connection errors to port 5173
