# Version 0.5.9 - Add Logout Feature

## Release Date
2026-03-09

## Summary
Fitur log out sudah tersedia sebelumnya namun memiliki masalah pada implementasinya. Tombol logout menggunakan Link dengan method GET yang tidak sesuai dengan route Laravel Fortify yang membutuhkan method POST. Perubahan ini memperbaiki implementasi logout agar berfungsi dengan benar.

## Problem
Sebelumnya, tombol logout di `user-menu-content.tsx` menggunakan komponen `<Link>` dengan prop `as="button"` yang mengirim request GET ke endpoint `/logout`. Namun, Laravel Fortify mengharuskan request POST untuk logout, sehingga fitur logout tidak berfungsi dengan benar.

## Solution

### 1. Frontend Changes

#### `resources/js/components/user-menu-content.tsx`

**Before:**
```tsx
const handleLogout = () => {
    cleanup();
    router.flushAll();
};

<DropdownMenuItem asChild>
    <Link
        className="block w-full cursor-pointer"
        href={logout()}
        as="button"
        onClick={handleLogout}
        data-test="logout-button"
    >
        <LogOut className="mr-2" />
        Log out
    </Link>
</DropdownMenuItem>
```

**After:**
```tsx
const handleLogout = (e: React.MouseEvent) => {
    e.preventDefault();
    cleanup();
    router.post(
        logout().url,
        {},
        {
            onFinish: () => {
                router.flushAll();
            },
        },
    );
};

<DropdownMenuItem
    onClick={handleLogout}
    data-test="logout-button"
    className="cursor-pointer"
>
    <LogOut className="mr-2 h-4 w-4" />
    Log out
</DropdownMenuItem>
```

### Key Changes:

1. **Remove `<Link>` component**: Tombol logout sekarang menggunakan `<DropdownMenuItem>` langsung tanpa `<Link>` wrapper
2. **Add `router.post()`**: Menggunakan `router.post()` dari Inertia untuk mengirim request POST ke endpoint logout
3. **Proper event handling**: Menggunakan `e.preventDefault()` untuk mencegah navigasi default
4. **Cleanup sequence**: 
   - Panggil `cleanup()` untuk menutup mobile navigation
   - Kirim POST request ke `/logout`
   - Setelah selesai, panggil `router.flushAll()` untuk membersihkan cache Inertia
5. **Improved icon styling**: Menambahkan `h-4 w-4` pada icon untuk konsistensi ukuran

## Technical Details

### How It Works

1. User clicks "Log out" menu item
2. `handleLogout` function dipanggil dengan event click
3. `e.preventDefault()` mencegah behavior default dropdown
4. `cleanup()` menutup mobile navigation jika terbuka
5. `router.post()` mengirim POST request ke `/logout` endpoint
6. Laravel Fortify menghandle request dan menghapus user session
7. Setelah selesai, `router.flushAll()` membersihkan cache Inertia
8. User di-redirect ke halaman login

### Route Information

- **Endpoint**: `POST /logout`
- **Controller**: `Laravel\Fortify\Http\Controllers\AuthenticatedSessionController@destroy`
- **Middleware**: Web middleware group (via Laravel Fortify)
