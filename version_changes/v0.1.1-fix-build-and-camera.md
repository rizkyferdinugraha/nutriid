# Version 0.1.1 - Fix Build and Camera Issues

## Date: 2026-03-04

## Changes

### 1. Fixed npm build error
- **File**: `resources/js/hooks/use-toast.ts` → `use-toast.tsx`
- **Issue**: File contained JSX syntax but had `.ts` extension
- **Fix**: Renamed to `.tsx` extension

### 2. Fixed database migration error
- **Files**: 
  - `database/migrations/2026_03_04_000001_create_food_scans_table.php`
  - `database/migrations/2026_03_04_000002_create_daily_summaries_table.php`
- **Issue**: `foreignUuid('user_id')` incompatible with `id()` in users table
- **Fix**: Changed `foreignUuid` to `foreignId` and `uuid('id')` to `id()`

### 3. Fixed models
- **Files**:
  - `app/Models/FoodScan.php`
  - `app/Models/DailySummary.php`
- **Issue**: Models used `HasUuids` trait but now use auto-increment IDs
- **Fix**: Removed `HasUuids` trait

### 4. Fixed AIService TypeError
- **File**: `app/Services/AIService.php`
- **Issue**: `$apiKey` property typed as `string` but could be null
- **Fix**: Changed to `?string` (nullable)

### 5. Fixed camera loading issue in scan page
- **File**: `resources/js/pages/scan.tsx`
- **Issues**:
  - Video stream not properly waiting for metadata before playing
  - Race conditions in camera initialization
  - Improper cleanup on unmount
- **Fixes**:
  - Added `onloadedmetadata` handler to wait for video readiness
  - Added `mounted` flag to prevent state updates after unmount
  - Added `cameraActive` state for proper camera restart
  - Improved cleanup in useEffect return

## Technical Notes
- Camera now properly initializes with `video.onloadedmetadata` event
- Uses `mounted` flag pattern to prevent memory leaks
- State management improved for camera restart functionality
