# v0.3.3 - Fix Duplicate Scan Bug

## Tanggal
2026-03-08

## Masalah
Saat scan berhasil, data disimpan ke database. Lalu jika user mengubah persentase konsumsi dan klik tombol "Simpan", data yang sama disimpan lagi (hanya beda consumed_percentage). Ini menyebabkan duplikasi data di database.

## Solusi
Memperbaiki logika di `resources/js/pages/scan.tsx` agar:

1. **Menyimpan scan ID**: Setelah analisis gambar berhasil, ID dari record yang dibuat disimpan di state `scanId`

2. **Update alih-alih Create baru**: Saat tombol "Simpan" diklik:
   - Jika `scanId` sudah ada, gunakan method PUT untuk UPDATE record yang sudah ada
   - Jika `scanId` belum ada, gunakan method POST untuk CREATE record baru

3. **Reset state**: Saat user klik "Scan Makanan Lain", semua state termasuk `scanId` di-reset

## Perubahan File

### `resources/js/pages/scan.tsx`
- Tambah state `scanId` untuk menyimpan ID dari record yang dibuat setelah analisis
- Modifikasi method `analyzeImage` untuk menyimpan `data.data.id` ke state `scanId`
- Modifikasi method `saveScan`:
  - Gunakan URL dinamis: `/api/food-scan/${scanId}` jika scanId ada, atau `/api/food-scan` jika belum
  - Gunakan method dinamis: PUT jika scanId ada, atau POST jika belum
  - Hanya kirim gambar jika scanId belum ada
  - Simpan pesan toast ke localStorage agar bisa ditampilkan di dashboard
- Modifikasi method `resetCapture` untuk mereset `scanId`

### `resources/js/hooks/use-toast.tsx`
- Tambah properti `duration` pada interface `Toast` untuk mendukung durasi kustom
- Modifikasi logic toast untuk menggunakan durasi dari props jika ada, default 5000ms jika tidak ada

### `resources/js/pages/dashboard.tsx`
- Import hook `toast` untuk menampilkan notifikasi
- Tambah logic di `useEffect` untuk mengecek localStorage dan menampilkan toast jika ada
- Toast ditampilkan dengan durasi 3 detik (3000ms) setelah redirect dari halaman scan

## Cara Kerja

### Flow Baru:
1. User scan gambar → `analyzeImage()` → POST ke `/api/food-scan` → Record dibuat dengan consumed_percentage = 0
2. Hasil ditampilkan dengan ID disimpan di `scanId`
3. User atur persentase konsumsi → klik "Simpan"
4. `saveScan()` deteksi `scanId` ada → PUT ke `/api/food-scan/{id}` → Update consumed_percentage
5. Redirect ke dashboard
